# automagic_in_n8n

Собрание готовых воркфлоу для n8n, упакованных по директориям с краткими описаниями и инструкциями по импорту и настройке учётных данных.

![logo](logo.png)

## Содержимое

-   **[01_google_api_docs](./01_google_api_docs/)**. AI-рецензент эссе. Автоматически проверяет текстовые документы, загруженные в Google Drive, и формирует структурированный отзыв с оценками по заданным критериям.

-   **[02_analysis_of_the_effectiveness_of_sales_managers](./02_analysis_of_the_effectiveness_of_sales_managers/)**. Автоматизирует сбор и анализ данных для оценки производительности менеджеров по продажам. Извлекает информацию из Airtable, рассчитывает KPI и обновляет итоговую статистику для выявления лучших и худших сотрудников.

-   **[03_Smart_Telegram_bot_for_personal_budget_management](./03_Smart_Telegram_bot_for_personal_budget_management/)**. Многофункциональный Telegram-бот для учёта личных финансов. Позволяет устанавливать лимиты, фиксировать расходы в свободной форме, запрашивать отчёты и получать AI-советы по оптимизации трат.

-   **[04_trend_researcher](./04_trend_researcher/)**. Автоматический исследователь трендов. Еженедельно сканирует интернет по заданным темам, отбирает релевантные материалы и генерирует сводный аналитический отчёт, который отправляется в `Telegram`.

-   **[05_flower_shop](./05_flower_shop/)**. Telegram-чатбот с ИИ для цветочного магазина. Автоматизирует консультации по ассортименту, отвечает на вопросы о доставке, оплате и помогает с оформлением заказов.

-   **[06_RAG_responses_supabase](./06_RAG_responses_supabase/)**. Демонстрирует создание и тестирование RAG-системы. Индексирует базу знаний из `Google Docs` в `Supabase`, а затем генерирует ответы на вопросы, используя эту базу.

-   **[07_Smart_License_Usage_Analyzer](./07_Smart_License_Usage_Analyzer/)**. Telegram-бот для управления виртуальными машинами и лицензиями. Позволяет создавать ВМ, отслеживать использование ресурсов (CPU/RAM), выявлять неактивные машины и управлять их удалением. Хранит данные в `PostgreSQL`.

-   **[08_github_issue_bot](./08_github_issue_bot/)**. Telegram-бот для работы с GitHub Issues на естественном языке. Через AI-агент (`OpenRouter`) позволяет создавать и просматривать issues голосовой или текстовой командой; персональный GitHub-токен пользователя хранится в `Redis`.

-   **[09_red_flags_bot](./09_red_flags_bot/)**. RAG-бот для юридического анализа договоров и пользовательских соглашений. Принимает текст или PDF-файл, разбивает на чанки, ищет совпадения в векторной базе `Qdrant` и возвращает структурированный список рисков с уровнем `HIGH / MEDIUM / LOW`. Подробная документация: https://trofimovelijah.github.io/red-flag-analysis/

## Быстрый старт

1.  Выберите интересующий воркфлоу из списка выше и перейдите в его директорию.
2.  Импортируйте файл `workflow.json` в ваш n8n: `Menu → Import from File`.
3.  Следуйте инструкциям из `README.md` в директории воркфлоу для настройки необходимых учётных данных (кредов).
4.  Активируйте воркфлоу и протестируйте его работу.

## Требования

-   Развёрнутый инстанс n8n (self-hosted или cloud).
-   Активные доступы/токены для внешних сервисов, используемых в воркфлоу (например, Telegram, AI-провайдеры, Airtable, Google Drive, OpenRouter и т.д.).

## Структура репозитория

-   `0X_workflow-name/` — директория с конкретной реализацией.
    -   `README.md` — описание и шаги по настройке воркфлоу.
    -   `workflow_name.json` — файл для импорта в n8n.
